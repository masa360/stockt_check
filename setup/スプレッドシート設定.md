# AnimePick-L セットアップガイド

このガイドの手順を上から順番に実行してください。  
所要時間の目安: **約30〜60分**

---

## STEP 1: スプレッドシートを作成する

1. [Google スプレッドシート](https://sheets.google.com) を開く
2. 「空白のスプレッドシート」を新規作成する
3. ファイル名を「AnimePick-L データベース」に変更する
4. 既に共有された作成リンクを使う場合は、そのまま新規作成してください
   - `https://docs.google.com/spreadsheets/create?usp=drive_web&ouid=109165756684285600874&folder=1uF0T6UW-qOEC_lceEMYk5U1VmmzkD0d0`

---

## STEP 2: GAS実行でシートを自動作成する（手動作成は不要）

このプロジェクトでは、シートを手で作る代わりに **`setupSheets()`** を1回実行して自動作成します。

1. スプレッドシートで「**拡張機能**」→「**Apps Script**」を開く  
2. `gas/Code.gs` の内容を丸ごと貼り付ける  
3. `SPREADSHEET_ID` を自分のシートIDへ変更  
4. 保存後、関数プルダウンで **`setupSheets`** を選択して実行  
5. 承認ダイアログを許可する  
6. 実行後、以下3シートが自動で作成されることを確認する

| 自動作成されるシート名 | 用途 |
|---|---|
| `商品マスタ` | 商品情報（JAN・名前・画像・棚番） |
| `指示書` | 当日の出荷指示リスト |
| `作業ログ` | 全操作の履歴（自動記録） |

---

## STEP 3: 各シートにヘッダー行を入力する

> 自動作成を使う場合、ヘッダーは自動で入るため読み飛ばし可能です。  
> 手動で確認したい人向けに列定義を残しています。

### 「商品マスタ」シート（1行目に入力）

| A列 | B列 | C列 | D列 | E列 | F列 | G列 |
|-----|-----|-----|-----|-----|-----|-----|
| JANコード | 商品名 | 画像URL | 棚番 | 単重(g) | 更新日時 | 更新者 |

### 「指示書」シート（1行目に入力）

| A列 | B列 | C列 | D列 | E列 | F列 | G列 |
|-----|-----|-----|-----|-----|-----|-----|
| 注文ID | JANコード | 商品名 | 必要数 | 完了数 | ステータス | 最終更新日時 | 最終更新者 |

> ※ H列に「最終更新者」も追加してください（上の表には8列あります）

### 「作業ログ」シート（1行目に入力）

| A列 | B列 | C列 | D列 | E列 | F列 | G列 | H列 |
|-----|-----|-----|-----|-----|-----|-----|-----|
| 日時 | LINE名 | LINEユーザーID | アクション | JANコード | 注文ID | 結果 | 備考 |

|--------|-----------|--------|--------|--------|------------|--------------|------------|

| ORD001 | 4901234567891 | （空欄でOK） | 2 | 0 | 未 | （空欄）| （空欄） |

## STEP 5: スプレッドシートのIDをコピーする

ブラウザのURLバーを見てください。

```
https://docs.google.com/spreadsheets/d/【ここがID】/edit
```

この「ここがID」の部分（長い英数字の文字列）をコピーしてください。

---

## STEP 6: Google Apps Script を設定する

1. スプレッドシートの上のメニューから  
   「**拡張機能**」→「**Apps Script**」をクリック
2. 左のファイル一覧に「コード.gs」があるのでクリック
3. 中身を**すべて削除**してから、`gas/Code.gs` の内容を**丸ごとコピペ**する
4. 1行目付近の以下の箇所を書き換える

```javascript
const SPREADSHEET_ID = 'YOUR_SPREADSHEET_ID'; // ← ここをSTEP5でコピーしたIDに変更
```

5. 保存ボタン（フロッピーアイコン）をクリック

---

## STEP 7: GAS をウェブアプリとしてデプロイする

1. 右上の「**デプロイ**」ボタン →「**新しいデプロイ**」をクリック
2. 「種類を選択」の歯車アイコン → 「**ウェブアプリ**」を選択
3. 以下のように設定する

| 項目 | 設定値 |
|------|--------|
| 説明 | AnimePick-L v1.0 |
| 次のユーザーとして実行 | **自分（自分のGoogleアカウント）** |
| アクセスできるユーザー | **全員（匿名ユーザーを含む）** |

4. 「**デプロイ**」ボタンをクリック
5. 「アクセスを承認」が出たら自分のGoogleアカウントで許可する
6. 表示された「**ウェブアプリのURL**」をコピーする  
   （`https://script.google.com/macros/s/XXXX/exec` という形式）

> ⚠ **重要**: コードを変更するたびに「**新しいデプロイ**」または「**デプロイを管理 → 編集**」で再デプロイが必要です

---

## STEP 8: LINE Developers でLIFFアプリを設定する

1. [LINE Developers](https://developers.line.biz/ja/) にアクセスしてLINEでログイン
2. 「プロバイダー」を作成（例: 会社名）
3. 「チャネル」を新規作成 → 「**LINE ログイン**」を選択
4. チャネルが作成されたら「LIFF」タブをクリック
5. 「**LIFFアプリを追加**」をクリック
6. 以下のように設定する

| 項目 | 設定値 |
|------|--------|
| LIFFアプリ名 | AnimePick-L |
| サイズ | **Full** |
| エンドポイントURL | Vercelで公開したURL（STEP 9参照）|
| スコープ | **profile** にチェック |
| Bot連携 | Off |

7. 「追加」ボタンをクリック
8. 表示された「**LIFF ID**」をコピーする  
   （例: `1234567890-AbCdEfGh` という形式）

---

## STEP 9: Vercelでデプロイする（推奨）

LIFF アプリとして動かすには、`liff/index.html` をHTTPS対応のURLで公開する必要があります。  
この手順では **Vercel** を使います。

1. GitHubリポジトリを使います  
   - `https://github.com/masa360/stockt_check.git`  
2. このプロジェクトの内容をリポジトリへ push する  
3. [Vercel](https://vercel.com) にGitHub連携でログイン  
4. 「Add New Project」→ `stockt_check` をImport  
5. Framework Preset は **Other** を選択（静的HTML）  
6. Root Directory はリポジトリ直下のまま  
7. Deploy を実行  
8. デプロイ完了後、`https://xxxxx.vercel.app/liff/index.html` を確認

> 補足: ルートを `liff/` にしたい場合は、Vercel Project Settings の Root Directory を `liff` に変更して再デプロイしてください。

---

## STEP 10: index.html に設定値を書き込む

`liff/index.html` をテキストエディタで開き、以下の2箇所を書き換えてください。

```javascript
const CONFIG = {
  LIFF_ID: 'YOUR_LIFF_ID',          // ← STEP8でコピーしたLIFF IDに変更
  GAS_URL: 'YOUR_GAS_WEBAPP_URL',   // ← STEP7でコピーしたGAS URLに変更
  ...
};
```

書き換えたら、GitHubへ push してVercelを再デプロイしてください。

---

## STEP 11: 動作確認

1. スマートフォンのLINEアプリを開く
2. LINE Developers の「LIFF URL」（`https://liff.line.me/【LIFF ID】`）にアクセス
3. 以下を確認する

- [ ] ログイン後に自分の名前が表示される
- [ ] 指示書に入力したサンプルが一覧表示される
- [ ] 「スキャン」ボタンを押すとカメラが起動する
- [ ] バーコードをスキャンすると商品名と「ピッキング完了」が表示される
- [ ] 完了数・ステータスが更新される
- [ ] 同じバーコードを10秒以内に再スキャンすると警告が出る
- [ ] 作業ログシートに操作が記録されている

## よくあるトラブル

| 症状 | 対処法 |
|------|--------|
| 「初期化に失敗しました」と表示される | LIFF IDが正しいか確認、LINE DevelopersのエンドポイントURLが正しいか確認 |
| 「通信エラー」が3回出る | GAS URLが正しいか確認、GASのデプロイが完了しているか確認 |
| スキャンしてもリストが更新されない | スプレッドシートのシート名が完全一致しているか確認（全角スペースに注意） |
| カメラが起動しない | HTTPSでアクセスしているか確認（HTTP不可）、カメラ許可をブラウザで許可 |
| 「対象の指示書が見つかりません」と表示される | 指示書シートのJANコードが商品マスタのJANコードと一致しているか確認 |
